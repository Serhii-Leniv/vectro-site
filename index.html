You can only babysit one terminal while it thinks. Vectro lets you start five, keep them from stepping on each other, and merge the one that got it right.

How it works

From a cold repo to merged work in three moves.

1

Open your repo

Point Vectro at any git project. Your canvas and terminals are saved per project — reopen and your agents come back up, not just empty shells.

2

Spin up agents

Each terminal runs your own CLI in its own branch + git worktree — up to nine, tiled automatically. Drag to reorder; the grid re-flows itself.

3

Review & merge

See who's working, idle, or waiting on you. Read each agent's diff and merge into your base — or discard the branch — without leaving the canvas.

A control room for parallel AI coding

Not another terminal emulator — a place to run a whole team of agents at once.

Parallel by design

A canvas of live terminals with automatic tiling. Start a swarm and watch them all at once — no tab-juggling.

True isolation

A dedicated branch and git worktree per agent, so parallel edits never clobber each other or your working tree.

Diff & merge in-app

Review an agent's changes against your base branch and land them — or throw the branch away — without breaking focus.

Knows who needs you

Live per-agent status — working, idle, or waiting for input — plus a desktop ping when a backgrounded agent finishes or gets stuck.

Picks up where you left off

Close the app and reopen later: each terminal restores in place and relaunches its agent, so you're back exactly where you stopped.

Yours to theme

A liquid-glass interface with your own accent colour, wallpaper, and adjustable terminal transparency.

Local & private

Runs on your machine, against your repo. No code leaves your computer — agents use your own keys.
